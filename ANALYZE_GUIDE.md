## 📋 详细使用步骤

### 第一步：抓取评论数据

1. 打开淘宝/天猫商品页面
2. 点击浏览器工具栏的 **"淘宝评论助手"** 插件图标
3. 点击 **"开始爬取评论"**
4. 等待抓取完成（显示评论数）
5. 点击 **"导出数据"**
6. 保存到 **Downloads** 文件夹

## 📊 报告内容详解

### 1️⃣ 营销文案报告

包含：
- **主标题**：基于用户好评提炼的核心卖点
- **副标题**：补充说明
- **核心卖点**：3-5个用户验证的卖点，附正面率
- **用户证言**：真实用户评价引用
- **推销话术**：开场白、需求挖掘、异议处理、促单话术

**使用场景：**
- 制作落地页
- 客服培训
- 广告投放文案

---

### 2️⃣ 产品深度分析报告

包含：
- **竞品差异化机会**：用户痛点 → 我方优势策略
- **产品迭代机会**：具体问题 → 改进建议（优先级排序）
- **用户赞誉亮点**：高频好评点
- **用户痛点反馈**：需要改进的问题

**使用场景：**
- 产品规划会议
- 竞品分析
- 改进优先级讨论

---

### 3️⃣ 分析报告摘要

包含：
- 核心卖点（一句话）
- 用户验证的5大卖点
- Top 3 改进建议

**使用场景：**
- 快速汇报
- 管理层决策参考

## 💡 分析原理

### 数据处理流程

```
原始CSV → 数据清洗 → 文本分析 → 机会挖掘 → 报告生成
   ↓          ↓           ↓           ↓           ↓
534条     去重过滤    关键词提取   差异化分析   营销文案
评论      有效450条   情感分析     改进建议     深度报告
```

### 核心算法

| 分析模块 | 技术方法 | 输出 |
|---------|---------|------|
| 关键词提取 | jieba分词 + TF-IDF | 高频关键词Top20 |
| 情感分析 | 基于词典 + 规则 | 正面/负面/中性 |
| 方面分析 | 关键词匹配 + 情感极性 | 6维度评价 |
| 机会挖掘 | 负面聚类 + 优先级排序 | 改进建议 |
| 文案生成 | 模板 + 用户证言填充 | 营销文案 |

## 🔧 常见问题

### Q1: 分析失败怎么办？

**检查：**
1. CSV文件是否正确导出？
2. 文件编码是否为UTF-8？
3. 数据量是否足够（建议>50条）？

**解决：**
```bash
# 手动指定文件路径
python3 analyze_reviews.py /path/to/your/评论数据.csv
```

### Q2: 如何分析多个商品？

```bash
# 分别分析每个商品
python3 analyze_reviews.py 商品A.csv
python3 analyze_reviews.py 商品B.csv

# 报告会分别保存
```

### Q3: 报告内容不准确？

**可能原因：**
- 评论数据量太少（<50条）
- 评论质量低（默认好评多）
- 商品品类特殊（需调整分析参数）

**优化建议：**
- 增加抓取评论数量
- 在 `src/config.py` 中调整 `MIN_COMMENT_LENGTH`
- 修改 `DEFAULT好评_PATTERNS` 过滤更多无效评论

## 🎓 进阶用法

### 自定义分析参数

编辑 `src/config.py`：

```python

MIN_COMMENT_LENGTH = 15  # 默认10

# 添加过滤词
DEFAULT好评_PATTERNS = [
    "默认好评",
    "评价方未及时做出评价",
    # 添加你的过滤词
]
```
