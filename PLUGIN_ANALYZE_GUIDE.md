# 🧩 插件一键分析功能使用指南

## ✨ 新功能介绍

插件 v1.1.0 新增**一键分析**功能，抓取评论后可直接保存到项目目录，快速生成分析报告！

---

## 🚀 使用流程

### 方式1：插件内一键保存（推荐）

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  插件抓取   │ → │  点击分析   │ → │  运行命令   │
│  评论数据   │    │  保存数据   │    │  生成报告   │
└─────────────┘    └─────────────┘    └─────────────┘
```

**步骤：**

1. **抓取评论**
   - 打开淘宝/天猫商品页
   - 点击插件"开始爬取评论"
   - 等待抓取完成

2. **一键保存**
   - 点击绿色的 **"📊 一键分析（生成报告）"** 按钮
   - 数据自动保存到 `output/crawler/` 目录
   - 插件显示保存的文件路径

3. **运行分析**
   - 复制插件显示的命令
   - 打开终端运行：
   ```bash
   python3 analyze_reviews.py "output/crawler/评论数据_xxx.csv"
   ```

4. **查看报告**
   - 报告生成在 `output/reports/` 目录
   - 包含：营销文案报告 + 深度分析报告 + 摘要

---

### 方式2：传统导出方式

如果你需要将数据保存到其他位置，使用传统的"导出数据"按钮。

## 🔧 技术说明

### 为什么需要手动运行命令？

由于浏览器安全限制，插件**无法直接运行 Python 脚本**。因此采用以下流程：

1. 插件保存 CSV 文件到项目目录
2. 显示需要运行的命令
3. 用户在终端运行命令生成报告

### 保存位置

一键分析保存路径：
```
ProductReviewAnalysis/output/crawler/评论数据_20260222_120000.csv
```

## 📋 常见问题

### Q: 点击"一键分析"后没有反应？

**检查：**
1. 是否已完成评论抓取？（必须先抓取数据）
2. 抓取的数据量是否为0？

### Q: 提示保存成功但找不到文件？

**检查：**
1. 浏览器下载设置是否允许自动下载
2. 项目目录权限是否正确

### Q: 如何修改保存位置？

**方法：** 使用传统"导出数据"按钮，手动选择保存位置。

### Q: 能否完全自动分析？

**答：** 由于浏览器安全限制，无法直接运行 Python。可以考虑：
- 使用 Native Messaging（需要安装本地应用）
- 使用 WebAssembly 重写分析逻辑（工作量大）

当前方案是平衡便利性和实现复杂度的最佳选择。

## 🎓 进阶用法

### 批量分析多个商品

```bash
# 抓取多个商品，分别保存
# 然后批量分析
for file in output/crawler/评论数据_*.csv; do
    echo "分析: $file"
    python3 analyze_reviews.py "$file"
done
```

### 自动监控新文件并分析

可以创建一个后台脚本，监控 `output/crawler/` 目录，有新文件时自动分析。
